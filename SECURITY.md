# Security Policy

Thank you for helping keep BlockyLauncher and its users safe.

## Supported Versions

We currently provide security fixes for the latest code on `main` and the most
recent tagged release.

| Version | Supported |
| --- | --- |
| `main` (unreleased) | ✅ |
| Latest release tag | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

Please **do not** open public GitHub issues for security reports.

Instead, report vulnerabilities privately by email:

- **security@blockymarketplace.com**

Include as much detail as possible:

- A clear description of the issue and impact
- Steps to reproduce
- A proof-of-concept, logs, or screenshots (if available)
- Affected commit/tag/version
- Any suggested mitigation

## Response Process

Our goals:

- Acknowledge your report within **3 business days**
- Provide an initial triage/update within **7 business days**
- Coordinate remediation and disclosure timeline with the reporter whenever
  possible

If a report is accepted, we will fix it in a private patch workflow, release
updates, and publish an advisory as appropriate.

## Scope

This policy covers vulnerabilities in this repository and released binaries for
BlockyLauncher.

For vulnerabilities involving external services (for example account systems or
APIs), include that context in your report and we will route internally.
